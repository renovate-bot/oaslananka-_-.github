# Governance

Repositories under this account are maintained by `oaslananka` unless a repository-local `GOVERNANCE.md` states otherwise.

## Decision model

Technical decisions are made by the maintainer based on repository scope, correctness, maintainability, security impact, and operational cost.

## Maintenance policy

- Keep default branches releasable.
- Prefer small, reviewable changes.
- Keep automation deterministic and documented.
- Remove unused workflows, dependencies, and templates instead of preserving historical clutter.
- Avoid broad automerge for package upgrades.

## Repository status

A repository may be archived when it is no longer maintained. Archived repositories should keep the latest useful documentation and security status visible.
