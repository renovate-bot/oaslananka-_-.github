# Support

Support is provided through GitHub issues and discussions when they are enabled for a repository.

## Use issues for

- Reproducible bugs.
- Documentation defects.
- Regression reports.
- Small feature requests with a clear use case.

## Use pull requests for

- Focused fixes.
- Documentation improvements.
- Test coverage additions.
- Dependency and CI maintenance when Renovate is not sufficient.

## Not supported

- Private consulting through public issue trackers.
- Requests unrelated to the repository scope.
- Reports without reproduction details when reproduction is required.
